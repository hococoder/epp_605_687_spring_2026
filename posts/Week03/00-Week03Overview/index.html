Welcome to Week 3! This week we'll cover how to work with data inside your app - from the smallest bits of data stored in UserDefaults to databases stored with Core Data and SwiftData.

  • Your App's Directory Structure: Learn about the underlying directory structure that makes up your app, where different types of files go, and how long they stick around.
  • Intro to User Defaults: Sometimes you just need to store a small amount of data - that's where User Defaults comes in.
  • Intro to the Codable Protocol: Learn how to encode and decode objects with a simple protocol declaration.
  • Introduction to SwiftData: iOS 17 introduced a branch new "Swifty" way to deal with databases with SwiftData. It's not a total replacement for Core Data yet, but it covers a lot of the basics!
  • Introduction to Core Data: For completeness, you'll learn how to use a powerful framework in iOS called Core Data to interface with an sqlite database under the hood. I'm pushing towards using SwiftData where possible, but this information about Core Data will let you use that if you wish.
  • Programming Assignment 3: In this assignment, you'll continue with the implementation of the database started in the lectures to make the ToDo App use that database when reading and writing data.

Don't forget! Development Log 2 is done at the end of this week!
