Overview

Let’s talk a bit about getting CloudKit enabled, how to setup schemas, and a few unique CloudKit features. As mentioned earlier, enabling CloudKit is pretty easy, and there are 2 steps to do it.

At Creation Time

First, you can choose to support CloudKit from the creation of your app. The screenshot below shows the new project dialog - if both "Use Core Data" (or SwiftData) and "Host in CloudKit" are selected, Xcode enables what it can in the source code when the project is created.

What exactly does it change in the source code? We'll take a look at that in the "Updating Your App for CloudKit" module later.

Setting up Capabilities

iCloud and Push Notifications

If you didn't choose "Host in CloudKit", and decide you want to add it later, you have to add the iCloud capability to your project. Under your project’s capabilities tab (under Project File -> Target -> Signing and Capabilities), click the "+Capability" button and search for and selected iCloud (which also adds Push Notifications for you, which is nice - you can get notifications from iCloud to respond to changes!).

When the display updates, Xcode will communicate with the Developer portal as well as the backend CloudKit servers to configure your project for CloudKit use.

If you chose "Host in CloudKit" when you made the Xcode project, these capabilities are added for you, and you just need to add containers.

Making a Container

At this point, you can make a new iCloud container for your app to use - give it the same name as your app bundle, but with a unique identifier behind it (if during your work, you need containers, please let me know - you need admin rights to make those containers, so only I can do it). A safe, unique, container can be something like what you see in the screenshot above - "iCloud." followed by the bundle identifier of your app (which above is "iCloud.edu.jh.ep.CloudKitDemo"). Again, Xcode will communicate with the back end to set everything up for you. You will also be able to see the other containers you have added under your development account. If for whatever reason things don’t happen automatically, you can force Xcode to try again with the “Fix Issues” button in the capabilities pane. If the container shows up colored red at creation, click the refresh button to sync things up with Apple's end.

Background Modes

As you see in the screenshot above, Xcode also enabled the "Remote Notifications" of the "Background Modes" capability, and added the "Push Notifications" capability:

You'll also notice there is an "CloudKit Dashboard" button - this opens up a web browser to give you a web view of the CloudKit databases you have access to. We'll come back to that in a minute.
