As with many powerful frameworks on iOS, URLSession has a delegate class, aptly named URLSessionDelegate, to handle particular events at the URLSession level such as:

  • Session invalidation (when your app no longer needs a session).
  • Receiving authentication challenges.
  • Notifying that a background session has finished.

    Methods from this class are usually paired with methods from its child classes - URLSessionTaskDelegate, URLSessionDataDelegate, and URLSessionDownloadDelegate, which we’ll discuss next.

    URLSessionDelegate Child Classes

    URLSessionTaskDelegate provides methods for task specific activities:

    • HTTP redirects
    • Authentication
    • Notifications when a data transmission in the task has finished.

      Something important to keep in mind - if you don’t explicitly make a delegate for your session, a system one is used, and you can get data from completions handlers.

      URLSessionDataDelegate provides methods specific to data and upload tasks:

      • Getting initial headers from the server.
      • Notifications for when data is received.
      • Whether data should be cached.
      • NOTE: This delegate should be used in coordination with URLSessionDelegate and URLSessionTaskDelegate.

      URLSessionDownloadDelegate provides methods specific to download tasks:

      • Notifying whether a download has resumed.
      • Informing about download progress.
      • Whether the download has finished.
      • NOTE: Again, this delegate should be used in coordination with URLSessionDelegate and URLSessionTaskDelegate.

      Don't forget - these delegate classes aren't needed, and instead you can use completion blocks (or async/await) to get the data you need - we'll see that in the example module later.
