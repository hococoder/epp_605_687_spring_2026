Since Swift is open source, it is in a constant state of evolution. In fact, Swift Evolution is the name of the process by which new proposals are made for Swift, reviewed by the community, and brought into the main branch. You'll often see new features referred to by their "SE" number (like SE-131), which is the Swift Evolution proposal number.

Swift 6.2

Swift 6.2 improvements centered around more approachable Concurrency syntax and defaults, and some new types, Span and inline arrays, to help with your everyday programming. You can read about those improvements here.

Swift 6.1

Swift 6.1 improvements mostly centered around concurrency improvements since Swift 6.0, mostly around the introduction of the nonisolated keyword.

Swift 6.0

One of the biggest updates for Swift 6 is the updates to concurrency (e.g. dealing with threads, protected data, etc). The starter projects will have Swift 6 language mode enabled, which will handle concurrency very strictly - you'll learn about some concurrency tips in a future module.

Some older additions before Swift 6.0

Let's take a look at some of the new features and syntactical sugar that have made their way into earlier versions of Swift.

Swift Macros

One of the biggest updates in Swift 5.9 had to be Swift Macros. Swift Macros allow you to change your code at compile time, letting you avoid repetitive code written by hand. This is always an additive process; you can only add code, never modify or delete it. The generated code is checked for syntactical correctness, and the values you pass into a macro are checked for type correctness.

There are 2 types: - Freestanding macros appear on their own, without being attached to a declaration. These are called with a pound/hashtag symbol - Attached macros modify the declaration that they're attached to. These are called with an @ symbol, like property wrappers. Yes, this is confusing.

Attached macros are seen everywhere in the new way to deal with databases, SwiftData. You'll learn more about that in a few weeks.

To read more about Swift Macros, please check out this set of docs at Apple

Clock improvements

Clocks, instants and durations are now easier to work with. You can simply write:

.now + .seconds(1)

where before you had to deal with nanoseconds.

You can also measure how long it takes things to happen:

let clock = ContinuousClock()

let duration = clock.measure {
  //Stuff to do here
}

Regex

Swift has a Regex type for dealing with string pattern matching:

let regex1 = /(.+?)ful day/
let phrase = "Wonderful day"
if let result2 = try regex1.wholeMatch(in: phrase) {
  print("Name: \(result2.1)")
}

These RegEx expressions are made from STRINGS! And they are checked for validity at COMPILE TIME! It's pretty amazing.

Optionally add modifiers with #if

Next week you'll learn about SwiftUI and modifiers which can be used to, well, modify those views. SwiftUI evolves pretty quickly as well, so you can now use #if to apply modifiers conditionally - which is useful if you have to change them based on iOS version or even platform:

Text("Hello 605.687!")
#if os(iOS)
    .font(.largeTitle)
#else
    .font(.headline)
#endif

Interchange CGFloat and Double

Not every framework has been moved to Swift yet, and there are still some pain points. One of those used to be when working with frameworks like CoreGraphics that use such unique types as CGFloat. Luckily, you can now interchangeable use CGFloat and Double types, which eliminates a lot of unnecessary casting.

Codable Synthesis with enums with associated values

It used to be hard to conform Enums with associated values to the Codable protocol, especially since they can get complicated:

enum Vehicle: Codable {
    case bike
    case car(cylinders: Int)
    case truck(horsePower: Double, carryingWeight: Double)
}

Before Swift 5.5 you had to make customized associated values for each type, but now Swift does this behind the scenes for you.

Conclusion

These are just some of the new features in earlier versions of Swift - we'll discuss a few others later in the semester.

In the next module, you'll learn how to start putting some of these Swift concepts into practice.
