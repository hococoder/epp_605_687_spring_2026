Built in Playground Macro

New in Xcode 26!

New in Xcode 26, the #Playground macro can be used to generate a playground in a Swift file - no special playground file required. The results show up in the same pane as your SwiftUI preview canvas, under a different tab.

#Playground {
  let x = 5
  let hello = "Hello"
  print("\(hello) \(x)")
}

Constants and Variables

let myName: String = "Josh Steele" //constant
var myAge = 45                     //can change

Swift promotes a "let first" concept when defining your properties - you should use let by default, and only use var when needed. This helps the compiler determine up front what can change in your app, and helps it catch cases where you try to modify constants elsewhere in your code.

Type Safety and Type Inference

Swift is a type-safe language. This means that the compiler knows at any given time the type of the values it is working with. This way, if you try to pass a String to an Int, the compiler can catch that as an error and tell you before runtime. The determination of type can happen in 2 ways:

  • Type Declarations. You clearly specific the type when declaring a value.
let myName: String = "Josh Steele"
  • Type Inference. The compiler looks at the value passed in and infers the type.
var myAge = 45  //inferred to be an Int

Because of this, I couldn't assign "Fourty-Five" to the variable myAge, since the compiler infers it is an Int, and cannot accept a String as a legitimate value.

Type inference can help determine a parameter type if declared during initialization:

let almostTheAnswer = 5*8.3

However, when parameters are defined separately, you will need to perform some casting:

let a = 5
let b = 8.3
//let c = a*b       //this won't compile if uncommented
let c = Double(a)*b

Special Operators

For the most part, Swift syntax is very similar to other modern languages like Java, Javascript, and Objective C. Here are a few syntactical differences

Closed Range Operator:

1...5

Half Open Operator:

1..<5

Repeat while:

var k = 3
repeat
{
  print("k is now \(k)")
  k = k - 1
} while k > 0

Oh and by the way.. no need for semicolons! (Did you notice?). if statements don't need parentheses (but you can put them in for readability):

var k = 3
if k < 4
{
    
}

Control Flow

for-in

Iterating over collections (see below) can be handled with a for-in statement:

let myArray = [1,2,3,4,5]
for myInteger in myArray
{
  print("The integer is \(myInteger)")
}

No classic c-style loop

Note that since Swift 3, the "c-style" for loop, for (int i=0; i<10; i++) is no longer valid! Instead, you can do something like this:

for i in 0..<10
{
  print(i)
}

This covers some of the syntactical differences you'll encounter in Swift. In the next several modules, we'll dive into some more advanced concepts in Swift that help it stand out from other languages.
