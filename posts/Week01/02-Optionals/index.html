Optionals

Page Contents

Optionals are a very important concept in Swift to master. They represent the possibility that a constant or variable may have a value OR it may be nil. Let's look at an example. If I have a class that stores a reference to a person's spouse's name, I might define it like this:

var spouseName: String = "Emily"

But what if the person doesn't have a spouse? I could set up all sorts of checks in my code, like adding an isMarried boolean and checking against that, but I don't have to do this because of optionals. I can simply make spouseName optional by putting a question mark after the type (Note I've given it a different name just so it doesn't collide with the declaration above):

var spouseName2: String?

If I look in the gutter after evaluating the playground, I see spouseName2 is set to nil. I can simply assign it if needed:

spouseName2 = "Emily"

I can also make it nil again by just setting the value to nil. I can also print out the optional:

print("Spouse is \(String(describing: spouseName2))")

As you can see, the value is still wrapped in the optional. What exactly does this mean? Well, let's think of an optional like this:

The optional may have a value inside - it may not. We don't know until it somehow gets unwrapped. Let's talk about how to do that next.

if-let

The if let statement peeks inside the optional box for you, and checks if the value is nil or not. If it isn't, the if statement succeeds, and executes what is in the block. Otherwise, the if statement fails:

if let spouseName2
{
  print("Spouse is \(spouse2)")
}

You can check multiple things in a if let statement - just separate them by commas.

Initializing Classes

Another important thing to know with optionals is that they can be used to help initialize classes. When a class initializes, all properties must have values assigned - and that can be nil. Let's take a look. If I have a computer class defined like this:

class Computer        //ERROR: Class 'Computer' has no initializers
{
  var brand: String
  var ramSize: Double
}

You can see the compiler complains if you uncomment that code. Each of the properties MUST be initialized. This can either be done through an init method (e.g. a constructor in Java) or by making them optional:

class Computer2
{
  var brand: String?
  var ramSize: Double?
}

This is a valid initialization - both values are set to nil. However, if I wanted to reference these values later on, I'd have to unwrap them every single time. If you know for a fact that the properties won't be referenced until AFTER you set their values, you can implicitly unwrap them at declaration time:

class Computer3
{
  var brand: String!
  var ramSize: Double!
}

This is also valid because you're telling the compiler "Look, I PROMISE these will be assigned to SOME value, possibly nil, before I use them." Just make sure to initialize it as promised, or you get an error when it tries to read the value.

One other side note about classes, initializers and optionals: Optional constants defined with the let keyword are allowed! They must be assigned in the initializer, but are fixed after that:

class Computer4
{
  let ramSize: Double?

  init(ramSize: Double)
  {
    self.ramSize = ramSize
  }
}
