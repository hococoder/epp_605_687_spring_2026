Let's walk through making a simple "Hello, world!" example using Xcode, Swift, and SwiftUI. See the video below for details.

Running in the Simulator:

For examples and assignments, we’ll use the built in Simulator in Xcode. The simulator can emulate many of the features of an actual device, with the exception of hardware specific items such as motion support, the proximity sensor, and the ambient light sensor to name a few. There are some other API restrictions that are listed at the URL on the slide. Simulators for the currently supported version of iOS come preinstalled in Xcode, and more can be added for testing older versions of iOS.

To run a project's app, choose a simulator from the drop down box next to the Scheme picker in the upper center portion of the Xcode user interface. Hit the play button in the upper left area to start it up! Note that you can choose various types of devices (including your Mac) and operating systems, so be sure to select the right simulator and iOS version before hitting play!

Running on Device:

Testing on devices is almost as easy as testing on a simulator. Simply hook up your device to the computer via USB, and the device becomes an option in the same drop down that contains the simulators. Testing on devices gives you a much more real world environment – for example, memory on a device is limited compared to what the Simulator has access to on your computer. Therefore you can see memory pressure problems occur on the device that were not apparent on the simulator. Before deploying to the App Store, be sure to test on a device (as many as you support, if possible!).

For this class, if you need to test on an actual device (e.g. for your final project), I can add your device as a test device to the developer portal if Xcode doesn't automatically do it. Just send me an email and we can walk through the process.
