Overview

The second protocol to adopt is AppIntent. This is the “verb” in the App Intents system structure or what you do with your entity. For the lesson example, an intent to open that lesson in your app might look like this to start:

import AppIntents

struct OpenLessonIntent: AppIntent, OpenIntent {

  static let title: LocalizedStringResource = "Open Lesson"

  @Parameter(title: "Lesson")
  var target: LessonEntity

  func perform() async throws -> some IntentResult {
    // a method to perform the navigation would go here
    return .result()
  }

  static var parameterSummary: some ParameterSummary {
    Summary("Open \(\.$target)")
  }
}

You have to adopt the AppIntent protocol, and if you want to make an intent that results in opening the app, you must also adopt OpenIntent. OpenIntent simplifies things a little by implementing some defaults in the protocol so you don't have to do it in your class.

The title property, which is just the title of this intent, and a perform() function help the adoption of the AppIntent protocol. The perform() function returns an IntentResult, or the result of performing this intent. Here, the result is empty, but it can pass back values if required. You can also do other things before you return, such as navigating to a particular screen. The parameter summary lets you show the user what the intent would do when it is, for example, shown in a Shortcut.
