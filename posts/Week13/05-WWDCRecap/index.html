Overview

The answer to "what didn't we cover from WWDC 2024" is A LOT. Apple's WWDC, or WorldWide Developer's Conference, is Apple's annual event where developers are the focus. In 2024, WWDC took place in an all virtual format, similar to what Apple has done since WWDC 2020, all thanks to COVID-19 (and Apple realizing they preferred to do things this way - which I also like!). All the platforms - iOS, macOS, watchOS, and tvOS - all get some sort of attention each year, where Apple employees guide you through the latest changes to the operating systems, so you can implement those new features in your app. There is WAY too much to cover here, so we'll cover a few things that I thought were high priority enough to get your attention, and leave the rest to learn about on your own. The Apple Developer App for macOS and iOS is a great way to watch the videos from many years of WWDC. I highly suggest it! For now, let's discuss the following:

  • if and switch statements can be used in assignments

There's not a ton in Swift that is high level (macros and some other low level stuff stole the show this year), but this is a neat one. The return values for both if statements and switch statements can now be directly assigned to properties:

let personType = if age < 18 { return "child" } else { return "adult" }

let personType = switch (age) {
  case 0..12: "preteen"
  case 12..17: "teen"
  case 18..110: "adult"
  default: "unknown"
}

Here the return types can be directly assigned to 'personType' depending on the passed in value. Not a huge change, but helpful in some cases.

  • Animating SF Symbols

In SwiftUI you can now animate SF Symbols:

Button {
    pressCount += 1
} label: {
    Label("Press me!", systemImage: "apple.logo")
}
.symbolEffect(.bounce, value: pressCount)
.font(.largeTitle)
  • Built in UI for In App Purchases

SwiftUI now has some built in views for showing IAP items in your app:

ProductView(id: "edu.jhu.rsteele3.subscription1") //shows 1 product
StoreView(ids: ["edu.jhu.rsteele3.subscription1", "edu.jhu.rsteele3.subscription2"]) //shows an array of products

And there are some others for subscriptions, and you can even customize the look and feel.

There are a TON of other changes in both Swift and SwiftUI, as Apple continually adds new things to the frameworks each year. Take a look through the WWDC 2023 videos for yourself and learn all about them!

The searchable() modifier

Inserting a search field, which seems like a no-brainer for certain apps, has always been less than easy to implement. That all changes in iOS 15 with the searchable() modifier:

ContentView()
  .searchable(text: $text)

It is available on all platforms, and can adapt accordingly based on what it is attached to. For example, when used with a NavigationStack it knows to place that search field on top of the entries being displayed.

NavigationStack {
  MyEntries(text: $text) {
    ForEach(data) { item in 
      Text("The item is \(item)")
    }
  }
}
.searchable(text: $text)

To tell if the user is actively searching, you can use the value of the \.isSearching keypath in the @Environment:

@Environment(\.isSearching) private var isSearching: Bool

This can let you update your display accordingly to display search results. Apple suggests using the .overlay modifier to put your results on top of the main list, so that main list remains unchanged.

Improvements to the Foundation Framework

The Foundation framework provides, well, a strong foundation for the rest of the Apple branded OSes out there. It contains classes for working with Dates, Strings, Collections, and more. Some notable improvements to Foundation this year were:

Support for Markdown

Mardkown was developed by John Gruber (of Daring Fireball) and Aaron Swartz in 2004. It has become a standard for its ability to convert "plain text" files with some markup into HTML. In Foundation, you can now apply style to your localized strings. For example:

In addition the API for AttributedString has been greatly improved, and both it and NSAttributedString from Objective C are now Localizable, like other strings in your app.

  • Foundation's formatters have been greatly improved! Instead of:
let dateFormatter = DateFormatter()

init() {
  dateFormatter.dateStyle = .medium
  dateFormatter.timeStyle = .medium
}

func generateMyData() {
  let formattedDate = dateFormatter.string(from: myDate)
  let formattedNumber = String(format: "%.2f", myNumber)
}

you can now do:

init() {
  //nothing here!
}

func generateMyData() {
  let formattedDate = myDate.formatted(date: .abbreviated, time: .standard)
  let formattedNumber = myNumber.formatted(.number.precision(.fractionalLength(2)))
}

This is a much more verbose way to do this formatting, it saves on some code (no more separate formatter initialization), and you lower your chances of getting the formatting string wrong when working with numbers.

Using DocC to document your code

DocC is a new, and open sourced, framework for not only documenting your code, but generating tutorials and reference documentation as well. The syntax is very straightforward, and supports Markdown:

/// The myFunc method
/// 
/// You can even include Markdown code blocks here
///
/// - Parameters: 
///   - foo: The Foo parameter
///   - bar: The Bar parameter
func myFunc(parameter1: Foo, parameter2: bar) {

}

The documentation can be built inside the Project -> Build Documentation menu, which also builds any tutorials or reference documents you have made outside your project.

The documentation is also immediately available to within Xcode - or a developer using your framework - by holding down alt and clicking on items such as parameters, classes and structs.

You can also export the documentation and deploy it to its own server, the engine for which Apple provides as part of DocC. Many of the modern day tutorials at Apple on SwiftUI are built and deployed using DocC!
